# Ablage-System UI Komponenten

Willkommen zum Storybook des Ablage-Systems. Hier finden Sie alle UI-Komponenten
dokumentiert und interaktiv testbar.

## Schnellstart

### Storybook starten
```bash
npm run storybook
```

### Tests ausführen
```bash
# A11y und Interaction Tests
npm run test:storybook

# Visual Regression Tests (Percy)
npm run test:visual
```

## Komponenten-Kategorien

### UI Komponenten
- **Button** - Alle Button-Varianten und -Größen
- **Card** - Container für gruppierte Inhalte
- **Alert** - Benachrichtigungen und Hinweise
- **DataTable** - Enterprise Data Table mit voller Funktionalität
- **Form** - Formular-Elemente (Input, Select, Checkbox, etc.)
- **Modal** - Dialog, AlertDialog und Sheet
- **Navigation** - Tabs, Breadcrumbs, Pagination

### Charts
- Line Chart
- Area Chart
- Bar Chart
- Pie/Donut Chart
- Waterfall Chart

## Design System

Das Ablage-System UI basiert auf:
- **shadcn/ui** - Wiederverwendbare UI-Komponenten
- **Tailwind CSS** - Utility-First CSS Framework
- **Radix UI** - Headless UI Primitives
- **Recharts** - Chart-Bibliothek

## Themes

Das System unterstützt zwei Themes:
- **Light Mode** (Standard)
- **Dark Mode**

Wechseln Sie das Theme über das Dropdown in der Storybook-Toolbar.

## Accessibility

Alle Komponenten werden automatisch auf Barrierefreiheit getestet:
- WCAG 2.0/2.1 Level A und AA
- Keyboard-Navigation
- Screen Reader Support
- Farbkontrast-Prüfung

## Responsive Design

Nutzen Sie die Viewport-Einstellungen um die Komponenten
in verschiedenen Bildschirmgrößen zu testen:
- Mobile (375px)
- Tablet (768px)
- Desktop (1280px)
- Widescreen (1920px)

## Beitragen

Neue Komponente hinzufügen:
1. Story-Datei erstellen: `src/stories/ComponentName.stories.tsx`
2. Alle Varianten als Stories exportieren
3. Dark Mode Story hinzufügen
4. Lokal testen: `npm run storybook`
5. PR erstellen - Percy läuft automatisch in CI
