## Kritische Bugs

### Bug 1: "NaN undefined" in Space-Cards

**Schweregrad:** Mittel
**Betroffene Komponente:** `PrivatSpaceList.tsx` / Space-Cards

**Beschreibung:**
In der Bereiche-Uebersicht werden Space-Cards mit "NaN undefined" angezeigt statt der korrekten Speichergroesse.

**Erwartetes Verhalten:**
Storage-Anzeige sollte "X MB / Y MB" oder aehnlich formatiert sein.

**Screenshot:** `tests/e2e/screenshots/04_bereiche_tab.png`

**Vermutete Ursache:**
- `formatBytes()` Funktion erhaelt undefined/null Wert
- API liefert kein `storage_used` oder `storage_limit` Feld
- Division durch undefined/null ergibt NaN

---

### Bug 2: Modul-Seiten zeigen "Fehler beim Laden..." ohne spaceId

**Schweregrad:** Kritisch (Blocking)
**Betroffene Komponenten:**
- `/privat/fahrzeuge` -> VehiclesPage
- `/privat/versicherungen` -> InsurancesPage
- `/privat/finanzen` -> FinancesPage

**Beschreibung:**
Wenn Benutzer auf Dashboard-Cards klicken (z.B. "Fahrzeuge"), werden sie zu `/privat/fahrzeuge` weitergeleitet. Diese Seite zeigt dann:

```
Fehler beim Laden der Fahrzeuge
```

**Root Cause:**
Die Routes sind OHNE spaceId definiert:
```tsx
// frontend/src/app/routes/privat.fahrzeuge.tsx
export const Route = createFileRoute('/privat/fahrzeuge')({
  component: VehiclesPage,
});
```

Die Pages erwarten aber eine spaceId:
```tsx
// VehiclesPage.tsx
const { spaceId: paramSpaceId } = useParams({ from: '/privat/spaces/$spaceId' });
const spaceId = propSpaceId || paramSpaceId; // paramSpaceId ist undefined!
```

**Betroffene Navigation:**
- PrivatDashboard.tsx verlinkt direkt auf `/privat/fahrzeuge` etc.
- Diese Routes haben KEINEN `$spaceId` Parameter

**Loesungsvorschlaege:**
1. **Option A:** Routes aendern zu `/privat/spaces/$spaceId/fahrzeuge`
2. **Option B:** Dashboard-Cards zeigen aggregierte Daten ALLER Spaces
3. **Option C:** Bei fehlendem spaceId ersten/Standard-Space automatisch waehlen

**Screenshot:** `tests/e2e/screenshots/06_fahrzeuge.png`
